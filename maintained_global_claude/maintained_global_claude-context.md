## Conventions

- This directory is the **edit target**, not `~/.claude/` directly. `setup.sh` symlinks everything here via `force_replace_targets` — edits directly in `~/.claude/` are overwritten on next `setup.sh` run.
- All hooks are Python scripts with uv inline dependencies (`#!/usr/bin/env -S uv run --script`). They receive JSON on stdin; exit non-zero to block the tool call.
- `defaultMode` is `"plan"` in `settings.json` — every session starts in plan mode. Skills listed in `permissions.allow` bypass confirmation prompts automatically.
- The entire `skills/` directory is a single directory-level symlink — new skill subdirectories are immediately available without updating `install_dotfiles`.

## Gotchas

- `pre_tool_use.py` blocks `rm -rf` variants and `.env` file access by exiting non-zero. If the hook has a Python syntax error, all tool calls will be silently blocked until fixed.
- `post_tool_use.py` logs to `.claude/logs/post_tool_use.json` relative to `cwd`, not `~/.claude/logs/` — logs appear in the active project directory, not here.
- `todos/`, `statsig/`, `history.jsonl`, `logs/`, `telemetry/`, `projects/`, `stats-cache.json` are Claude Code runtime state written into `~/.claude/` — not configuration, not version-controlled. They appear in this repo's working tree because this directory IS `~/.claude/` via symlinks.
