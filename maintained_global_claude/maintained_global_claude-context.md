## Conventions

- This directory is the **edit target**, not `~/.claude/`. Changes made directly in `~/.claude/` will be lost on next `setup.sh` run. The `force_replace_targets` array in `install/install_functions.sh` always overwrites `~/.claude/` from here.
- `CLAUDE.md` here is the global Claude instructions file (same content as `~/.claude/CLAUDE.md`). It is loaded for every Claude Code session across all projects.
- Hooks are Python scripts invoked by the harness ‚Äî they must be runnable via `uv run ~/.claude/hooks/<script>.py`. They live in `hooks/` with utilities in `hooks/utils/`.
- `defaultMode: plan` means Claude Code always starts in plan mode ‚Äî agents must call `ExitPlanMode` permission to act. This is intentional and should not be changed without updating `settings.json`.

## Gotchas

- `todos/`, `statsig/`, `history.jsonl`, `logs/`, `telemetry/`, `projects/`, `session-env/`, `debug/`, `downloads/`, `stats-cache.json` are Claude Code runtime state ‚Äî not tracked in git and not part of the config. They accumulate in `~/.claude/` and appear here because this dir IS `~/.claude/` (via symlinks).
- Skills with subdirectories (e.g., `skills/create-plan/`) are symlinked as directories, not individual files. Adding a new skill subdir requires updating `install_dotfiles` in `install/install_functions.sh`.
- `context7@claude-plugins-official` and `skill-creator@claude-plugins-official` are the only enabled plugins ‚Äî all others are explicitly `false`.
